# aurem-cto — ORA CTO Sovereign (Legion Standalone)

> Hybrid Standalone deployment of ORA CTO for the founder's Legion laptop.
> Survives Emergent platform outages. Shares Atlas MongoDB with `aurem.live`.

## Architecture

```
┌────────────────────────────┐         ┌────────────────────────────┐
│  AUREM Customer (Emergent) │  HTTPS  │   ORA CTO (Legion laptop)  │
│  https://aurem.live        │ ◄─────► │   https://cto.aurem.live   │
│  FastAPI + React           │         │   FastAPI + React + SQLite │
│  Stripe, leads, audits     │         │   28 ORA tools + watchdog  │
└────────────────────────────┘         └────────────────────────────┘
            ▼                                        ▼
            └──────────► Atlas MongoDB ◄─────────────┘
                         (shared, primary + outbox replay)
```

## Quick Start (on Legion laptop)

```bash
# 1. Copy this folder to the laptop
scp -r aurem-cto/ legion:/opt/

# 2. Install docker + cloudflared if missing
ssh legion 'cd /opt/aurem-cto && sudo bash bootstrap.sh'

# 3. Fill in .env (use .env.example as template)
ssh legion 'cd /opt/aurem-cto && cp .env.example .env && nano .env'

# 4. Bring up the stack
ssh legion 'cd /opt/aurem-cto && docker compose up -d'

# 5. Verify
curl https://cto.aurem.live/api/health
# → {"ok":true,"service":"ora-cto-sovereign","uptime_s":42}
```

## Cloudflare Tunnel (required for cto.aurem.live)

```bash
# One-time setup
cloudflared tunnel login
cloudflared tunnel create aurem-cto-legion
cloudflared tunnel route dns aurem-cto-legion cto.aurem.live

# Config is in /etc/cloudflared/config.yml (see cloudflared/config.yml.example)
sudo systemctl enable --now cloudflared
```

## Outbox Pattern

Every write that mutates state lands in a local SQLite table `outbox_pending`
first.  The `outbox` worker polls that table every 5 seconds and pushes each
row to Atlas MongoDB.  This means:

- Atlas down? ORA CTO keeps working. Writes queue locally.
- Legion offline? AUREM stays running on Atlas.
- When connectivity returns, the worker replays pending writes.

## Environment Variables

See `.env.example` for the full list.  Required:

- `MONGO_URL` — Atlas connection string (shared with aurem.live)
- `DB_NAME` — usually `aurem_db`
- `EMERGENT_LLM_KEY` — for Claude Sonnet 4.5 fallback
- `GROQ_API_KEY` — primary LLM
- `JWT_SECRET` — must match the upstream AUREM `JWT_SECRET` for token interop
- `CTO_PUBLIC_URL` — usually `https://cto.aurem.live`
