§ 05 · Observability & Self-Healing

What watches itself.

AUREM includes its own triple-layer observability stack — unique among SMB-focused SaaS platforms. Every user-facing error, API failure, and endpoint degradation is captured, classified, and (where mechanically safe) auto-healed without human intervention.

SystemScopeAuto-HealAI Layer
QA Bot · Layer 120 critical endpoints pinged every 10min—Alert via Resend (recurring failures)
QA Agent · Layer 2Chained user journeys, weekly deep run—Claude 4.5 RCA on failures
Sentinel ClientAll browser-side JS errors + API failures✓ Stale URL · ChunkLoad · 401 · 429Claude 4.5 structured repair suggestions
Site MonitorCustomer-facing uptime checks (5-min cadence)✓ Incident lifecycle automationEnterprise tier includes AI RCA
◆ Trust-but-Verify Repair Model
The Sentinel Client Dashboard uses a 3-tier model: (1) silent capture, (2) mechanical auto-heal for known patterns, (3) Claude-powered repair suggestions that are never auto-applied. Every code change remains a human decision via the Admin Review queue. This is the differentiator vs Sentry: AUREM diagnoses; humans approve.
